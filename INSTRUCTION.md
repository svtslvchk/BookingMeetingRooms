# Руководство по запуску и проверке проекта

Этот документ содержит пошаговое руководство по локальному развертыванию, настройке окружения, запуску тестов и ручной проверке всех функций системы бронирования комнат.

---

## Шаг 1. Подготовка окружения и переменных (`.env`)

В корневой директории проекта создайте файл `.env`. Он необходим для того, чтобы backend и Alembic могли подключаться к базе данных и использовать настройки безопасности JWT.

Заполните его следующим содержимым:

```env
# Настройки подключения к базе данных PostgreSQL
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=your_secure_password
DB_NAME=booking_db

# Настройки безопасности JWT
# Можно сгенерировать командой:
# openssl rand -hex 32
JWT_SECRET_KEY=super_secret_random_hex_string_here
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
```

---

## Шаг 2. Установка зависимостей

Убедитесь, что установлен **Python 3.12** или новее.

Создайте виртуальное окружение и установите зависимости:

```bash
# Создание виртуального окружения
python3 -m venv .venv

# Активация (Linux / macOS / WSL)
source .venv/bin/activate

# Windows PowerShell
.venv\Scripts\Activate.ps1

# Windows CMD
.venv\Scripts\activate.bat

# Обновление pip
pip install --upgrade pip

# Установка зависимостей
pip install -r requirements.txt
```

---

## Шаг 3. Создание базы данных и применение миграций

Убедитесь, что сервер PostgreSQL запущен и существует пустая база данных с именем, указанным в `.env` (например, `booking_db`).

Примените все миграции:

```bash
alembic upgrade head
```

После выполнения команды структура базы данных будет создана автоматически.

---

## Шаг 4. Автоматическая проверка (тесты)

Перед запуском приложения рекомендуется убедиться, что вся бизнес-логика работает корректно.

Запустите тесты:

```bash
pytest -v
```

> **Важно**
>
> Тесты используют отдельную тестовую базу данных (настраивается автоматически в `tests/conftest.py`), поэтому данные основной базы не будут изменены.

---

## Шаг 5. Запуск приложения

Запустите сервер разработки:

```bash
uvicorn app.main:app --reload
```

После запуска приложение будет доступно по адресу:

```
http://127.0.0.1:8000/
```

# Ручная проверка функциональности

## 1. Веб-интерфейс

Адрес:

```
http://127.0.0.1:8000/
```

### Сценарий проверки

1. Зарегистрируйте нового пользователя.
2. Выполните вход.
3. Убедитесь, что JWT-токен сохранился в браузере.
4. Создайте тестовую комнату.
5. Создайте бронирование этой комнаты.

---

## 2. Swagger UI

Адрес:

```
http://127.0.0.1:8000/docs
```

Интерактивная документация позволяет протестировать все REST API.

### Сценарий проверки

Попробуйте отправить запрос с некорректными данными.

Например:

- дата окончания бронирования меньше даты начала.

Ожидаемый результат:

- HTTP **422 Unprocessable Entity**;
- структурированный JSON с описанием ошибки валидации.

---

## 3. Проверка защиты от пересечения бронирований

Создайте бронирование:

| Комната | Начало | Конец |
|---------|---------|--------|
| №1 | 12:00 | 13:00 |

После этого попробуйте создать второе бронирование:

| Комната | Начало | Конец |
|---------|---------|--------|
| №1 | 12:30 | 13:30 |

### Ожидаемый результат

Сервер должен отклонить запрос и вернуть:

- HTTP **409 Conflict**;
- JSON с сообщением о конфликте бронирований.

Это подтверждает корректную работу механизма защиты от пересечения интервалов бронирования.
