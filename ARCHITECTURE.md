# Архитектурный паспорт проекта (Design Document)

Данный документ описывает внутреннее устройство, архитектурные решения и модель данных системы бронирования переговорных комнат.

## 🏛️ Компонентная архитектура

Приложение спроектировано по классической трехслойной архитектуре (Layered Architecture):

1. **слой представления (API Routes):** Принимает HTTP-запросы, валидирует входящие данные с помощью Pydantic-схем и вызывает нужные сервисы.
2. **Слой бизнес-логики (Services):** Чистая магия приложения. Здесь живет логика проверки пересечений интервалов времени и бизнес-правила.
3. **Слой доступа к данным (Repositories):** Инкапсулирует в себе сырые SQL-запросы и операции ORM SQLAlchemy. Позволяет легко заменить базу данных при необходимости.



## 🗄️ Модель данных и база данных

В качестве СУБД выбран PostgreSQL. За счет использования индексов достигнута высокая скорость работы:
- **Таблица Users:** Хранит хэшированные пароли (bcrypt). Исключено хранение в открытом виде.
- **Таблица Rooms:** Переговорные комнаты.
- **Таблица Bookings:** Связующая сущность с составным индексом `(room_id, time_start, time_end)`. Это позволяет базе данных выполнять поиск наложения интервалов за O(log N) вместо полного сканирования таблицы.

## 🔐 Протокол безопасности (Auth Flow)

В системе реализована бессессионная (Stateless) авторизация на базе JWT (JSON Web Tokens):
1. Клиент отправляет логин/пароль на `/auth/login`.
2. Сервер проверяет хэш и возвращает `access_token` с ограниченным сроком жизни (например, 30 минут).
3. Клиент сохраняет токен в `localStorage` и прикрепляет его к каждому запросу в заголовке `Authorization: Bearer <token>`.
4. Кастомные зависимости FastAPI (`Depends`) перехватывают токен, валидируют его и автоматически прокидывают текущего пользователя в контроллер бэкенда.

## ⚡ Обработка ошибок и отказоустойчивость

- Любое необработанное исключение (500 Error) перехватывается глобальным `Exception Handler`, который пишет подробный лог (Traceback) во внутреннюю систему логирования сервера, а клиенту отдает безопасный структурированный JSON без утечки внутренних данных.
- База данных полностью изолирована внутри Docker-сети и недоступна из внешнего мира напрямую — все запросы идут строго через бэкенд FastAPI.
